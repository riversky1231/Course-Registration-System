### 3. 八重校验逻辑详解

#### 校验执行顺序

```
选课请求
  ↓
【已有校验 1-3】
  ├─ 1. 重复选课检测 ✓
  ├─ 2. 课程容量检测 ✓
  └─ 3. 时间冲突检测 ✓
  ↓
【新增校验 4-8】🆕
  ├─ 4. 年级权限校验
  ├─ 5. 先修课程校验（支持链式）
  ├─ 6. 互斥课程校验（双向）
  ├─ 7. 课程类型限选校验
  └─ 8. 学分上限校验（GPA动态）
  ↓
选课成功
```

#### 校验4：年级权限校验

**逻辑**: 低年级学生不能选高年级课程

**示例**:
```
学生：大二（grade=2）
课程：微服务架构（grade_limit=4）
结果：❌ 该课程仅限大4及以上年级学生选修，你当前是大2
```

---

#### 校验5：先修课程校验（最复杂）

**逻辑**: 必须先修完前置课程且成绩达标

**支持链式依赖**:
```
微服务架构 (需要65分)
  ↓
分布式系统 (需要60分)
  ↓
数据库原理 (需要60分)
  ↓
数据结构
```

**示例**:
```
学生想选：微服务架构
系统检查：
  ✓ 已修 分布式系统？成绩70分 ≥ 65分 ✓
    ✓ 已修 数据库原理？成绩85分 ≥ 60分 ✓
      ✓ 已修 数据结构？成绩90分 ≥ 60分 ✓
结果：✅ 通过
```

**失败示例**:
```
学生想选：软件架构设计
系统检查：
  ✓ 已修 Java Web开发？成绩65分 < 70分 ❌
结果：❌ 先修课程《Java Web开发》成绩需达到70分（当前：65分）
```

---

#### 校验6：互斥课程校验

**逻辑**: 已选其中一门课程后，不能再选另一门

**双向关系**:
```
Java Web开发 ↔ Python Web开发
（选了其中一门，就不能选另一门）
```

**示例**:
```
学生已选：Java Web开发
学生想选：Python Web开发
结果：❌ 你已选修《Java Web开发》，与当前课程互斥，不能同时选修
     （Java Web开发 与 Python Web开发 为同类型课程，只能选其一）
```

---

#### 校验7：课程类型限选校验

**逻辑**: 某类型课程最多只能选N门

**配置示例**:
- 通识选修：最多3门
- 专业选修：最多5门

**示例**:
```
学生已选通识选修：艺术鉴赏、创新创业、音乐欣赏（3门）
学生想选：大学英语（通识选修）
结果：❌ 《通识选修》类课程最多选3门，你已选3门
```

---

#### 校验8：学分上限校验（GPA动态调整）

**逻辑**: 根据学生GPA确定本学期可选学分上限

**GPA分级规则**:
| GPA范围 | 最大学分 | 说明 |
|---------|---------|------|
| 0.0 - 2.0 | 15学分 | 学业预警 |
| 2.0 - 3.0 | 20学分 | 正常 |
| 3.0 - 3.5 | 25学分 | 优秀 |
| 3.5 - 4.0 | 30学分 | 学霸特权 |

**示例**:
```
学生GPA：3.2
系统判定：3.0 ≤ 3.2 < 3.5 → 最多25学分
学生已选：18学分
学生想选：8学分的课程
计算：18 + 8 = 26 > 25
结果：❌ 本学期最多可选25学分（GPA 3.20），当前已选18.0学分，
     加上该课程8学分将超出上限
```
