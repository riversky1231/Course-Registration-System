## 2️⃣ 白盒测试支持度：⭐⭐⭐⭐⭐

### 2.1 推荐测试函数：`CourseValidationService.validatePrerequisites()`

这是系统中**最复杂**的函数，包含：
- 循环结构（遍历先修课程列表）
- 多重条件判断（是否已修、是否录入成绩、分数是否达标）
- 异常抛出（3个不同的错误分支）

#### 源代码（简化版）：

```java
public void validatePrerequisites(String courseId, String studentId) {
    List<CoursePrerequisite> prerequisites = prerequisiteMapper.selectWithNamesByCourseId(courseId);

    if (prerequisites.isEmpty()) {  // 判定1
        return;
    }

    for (CoursePrerequisite prereq : prerequisites) {  // 循环
        SelectionRecord completed = selectionMapper.selectOne(...);

        if (completed == null) {  // 判定2
            throw new AppException("需要先修完《" + prereq.getPrerequisiteCourseName() + "》");
        }

        if (!Boolean.TRUE.equals(completed.getGraded())) {  // 判定3
            throw new AppException("先修课程《" + prereq.getPrerequisiteCourseName() + "》成绩尚未录入");
        }

        if (completed.getScore() == null || completed.getScore() < prereq.getMinScore()) {  // 判定4
            throw new AppException("先修课程《" + prereq.getPrerequisiteCourseName() + "》成绩需达到" + prereq.getMinScore() + "分");
        }
    }
}
```

### 2.2 控制流图（CFG）

```
        [开始]
           ↓
    [1] 获取先修课程列表
           ↓
    [2] 判定1: prerequisites.isEmpty()?
           ↓ N          ↓ Y
    [3] 进入循环      [10] return (结束)
           ↓
    [4] 查询学生选课记录
           ↓
    [5] 判定2: completed == null?
           ↓ N          ↓ Y
    [6] 判定3: graded?  [11] 抛异常1 (结束)
           ↓ N          ↓ Y
    [7] 判定4: score达标? [12] 抛异常2 (结束)
           ↓ N          ↓ Y
    [13] 抛异常3 (结束)  [8] 继续循环
           ↓
    [9] 循环结束 → return (结束)
```

**节点编号**：
- 节点1: 入口
- 节点2: 判定1（是否有先修课程）
- 节点3-8: 循环体
- 节点5: 判定2（是否已修）
- 节点6: 判定3（是否录入成绩）
- 节点7: 判定4（分数是否达标）
- 节点9-13: 出口节点

### 2.4 路径覆盖（Path Coverage）

**独立路径数**: 根据McCabe环路复杂度计算

```
V(G) = E - N + 2P
```

其中：
- E = 边数 = 15
- N = 节点数 = 13
- P = 连通分量数 = 1

```
V(G) = 15 - 13 + 2×1 = 4
```

**独立路径**：

1. **路径1**: 1→2→10（无先修课程）
2. **路径2**: 1→2→3→4→5→11（未修先修课程）
3. **路径3**: 1→2→3→4→5→6→12（先修课程未录入成绩）
4. **路径4**: 1→2→3→4→5→6→7→13（先修课程分数不达标）
5. **路径5**: 1→2→3→4→5→6→7→8→9（全部满足）

**测试用例**: 需要 **5个用例** 覆盖所有独立路径
